Hardcoded Searches

Setup some values for the user, let them search on some others. The apple pie of searches.

Just as with the hardcoded searches and basic example we can sift out results based on cell values. This time we'll combine both a search form, and hardcoded paramters to get a neat little dynamic filtering, with some built in personalization.

For this, a simple search form will suffice:

{exp:sift:form
	matrix_field="product_options"}
	<input type="text" name="cell_one"/>
	<input type="submit" value="Sift"/>
{/exp:sift:form}

Along with a results view with some values passed:

<h1>Your Personal Options</h1>
{exp:sift:view
	matrix_field="product_options"
	sizes="Medium"
	gender="male"}
	<h2>{title}</h2>
	<ul>
	{product_options}
		<li>{cell_one}, Size : {sizes}, Gender : {gender}</li>
	{/product_options}
	</ul>
{/exp:sift:view}

Now when a user searches via the search form all the search params will be used. Neat.

Note: Passed search terms via GET or POST will take precedence over paramters on the sift:view tag. You can use this to setup default filters that can be overridded by a user.

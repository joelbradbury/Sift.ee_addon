Basic Example

Simply using the sift:form and sift:results tags to build the simplest of matrix searches.

Showing the Search Form

{exp:sift:form
	matrix_field="product_options"}
	<input type="text" name="cell_one" value="{cell_one}/>
	<input type="submit" value="Sift"/>
{/exp:sift:form}

Will give you something like this :

<form class="form-search" method="get" action="/search">
	<input type="text" name="cell_one" value=""/>
	<input type="submit" value="Sift"/>
</form>

The value tag will be replaced with the previously passed search term, if there was one passed, or a blank string otherwise. The other half of the setup is the view tag:

Showing the Results

<h1>Sift Results</h1>
{exp:sift:view matrix_field="product_options"}
	<h2>{title}</h2>
	<ul>
	{product_options}
		<li>{cell_one}</li>
	{/product_options}
	</ul>
{/exp:sift:view}

Will give you something like this :

<h1>Sift Results</h1>

<h2>Entry One</h2>
<ul>
	<li>Matrix Cell Value One</li>
	<li>Matrix Cell Value Two</li>
	<li>Matrix Cell Value Three</li>
</ul>

<h2>Entry Two</h2>
<ul>
	<li>Matrix Cell Alternative One</li>
	<li>Matrix Cell Alternative Three</li>
</ul>

..

Note: Notice that the in the results view only the matrix rows that match the search terms are returned. They've been sifted if you will.

That's all there is too it! The resulting rows in the matrix field displayed are only those that match the passed search term.

Note: You can search on all the columns in a matrix field at the same time. Simply pass the columns as seperatly named inputs. Searches are performed inclusively.

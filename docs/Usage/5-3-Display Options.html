Display Options

Switch up the results display.

Wouldn't things be boring if we didn't switch up the display once in a while? The sift:view tag gives all the control of the channel:entries tag, but with some tweaks. There's the Standard View and more the interesting Seperated View.

Switching between the views

By default sift uses the standard view. To switch to the seperated view pass the parameter 'seperated_view="yes"' on the sift:view tag, or as an input of the same name in the sift:form tag.

Standard View

Have you used the channel:entries tag? Yes? Good. Then you know what the standard view is like. All the same tags on the channel:entries tags will work on the sift:view tag, and the output is broadly the same.

The one change is the way the matrix rows are sifted down. Not only does sift filter down the entries returned, it also removes any matrix rows from the results that don't match your sift search.

The upshot of this is - the matrix loop simply doesn't know about any non-matching rows. To your matrix field, it's as if they don't exist. Great if you're wanting to limit the view on a product to only relevant options.

Seperated View

The seperated view is slightly different. This time, instead of simply showing entries and row's that match the search, each matching matrix row behaves as it's own entry. What? I know, crazy right?

Note: The seperated view can be quite memory intensive when the results set is large. When dealing with lots of entries, with lots of matrix rows, things quickly add up. ie. 50 results, with 20 rows each, is 1000 seperated entries. To keep things performant the seperated view is internally limited to only be avaialble when the result set contains 25 or less items. When above this limit, the standard view will be used.

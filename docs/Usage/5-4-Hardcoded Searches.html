Hardcoded Searches

Bypass the search form, and choose the sifting parameters directly. Maybe based on a member's preferences?

Perhaps you've got some data about a user available to you in your templates. Instead of showing a normal search form and sifting from that, you can supply all the same values as parameters on the sift:view tag.

For example:

<h1>Your Personal Options</h1>
{exp:sift:view
	matrix_field="product_options"
	sizes="Medium"
	gender="male"}
	<h2>{title}</h2>
	<ul>
	{product_options}
		<li>{cell_one}, Size : {sizes}, Gender : {gender}</li>
	{/product_options}
	</ul>
{/exp:sift:view}

This would be identical to if a user searched with the sift:form tag for sizes="Medium" and gender="male". So it would return similar to:

<h1>Your Personal Options</h1>
<h2>Man's Shirt</h2>
<ul>
	<li>Blue, Size : Medium, Gender : Male</li>
	<li>Green, Size : Medium, Gender : Male</li>
	<li>Red, Size : Medium, Gender : Male</li>
</ul>

<h2>Man's Jumper</h2>
<ul>
	<li>Check, Size : Medium, Gender : Male</li>
	<li>Plaid, Size : Medium, Gender : Male</li>
</ul>
..

Note: You can combine this with a normal search via a form too! Check out the Merged Searches example.

Basic Setup

Setting up Sift on your site is super simple. The basic usage consists of two simple parts:

  1. Displaying a Search Form
  2. Showing the Results View

Note: We'll use the field name of 'product_options' throughout these examples. This is just a standard Matrix field, with a few columns

Briefly :

1. Displaying a Search Form

The search form is a very simple setup. 

{exp:sift:form
	matrix_field="product_options"}
	<input type="text" name="cell_one"/>
	...
	<input type="submit" value="Sift"/>
{/exp:sift:form}

Within the exp:sift:form tag pair simply add form fields with the same name as the matrix columns. In the above example, we've got an input called 'cell_one', which corresponds to a column in the 'product_options' matrix field of the same name.

There are a few options available when using the {exp:sift:form} tag. Checkout the full :form tag documentation.

Note: You can setup your own form without using this tag. Simply make sure you have a hidden field called 'sift_search' with a value of 'yes'

2. Showing the Results View

The results view tag can be used in the same was as a normal channel:entries tag :

{exp:sift:view}
	{title}
	{product_options}
		{cell_one}
	{/product_options}
{/exp:sift:view}

Here we're just simply looping over the resulting entries, and the matrix field exactly as you would in a normal channel:entries tag. Simple as pie! Sift will automatically limit both the entries returned, and the matrix rows returned, to just those that match your search.

The sift:view tag can do some interesting things. It can even split up results in matrix rows, so that if multiple rows from the same entry match your search, it will loop over each as if they were from seperate entries, instead of only looping over the matrix field tag pair. For more info, check out the full tag documentation.

Going Further

Sift has lots more options to let you control the searching and output. Have a look through the examples to get an idea of what you can do with it.
